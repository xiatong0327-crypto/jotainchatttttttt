# jotainchatttttttt — Design Document

**Date:** 2026-07-14  
**Status:** Decisions locked; Mac-only scope; architecture reviewed  
**Scope:** **Mac ↔ Mac only** — LAN chat + basic photo/file transfer. No Windows track.

---

## 1. Problem

On the same Wi‑Fi / LAN, day-to-day communication is awkward for large files. Using third-party IMs sends data through the cloud and increases breach / retention risk. AirDrop is not a full chat product and may not fit every workflow.

**jotainchatttttttt** is a local-only **macOS** desktop app: discover peers on the LAN, 1:1 chat, and transfer photos/files via the Wi‑Fi router — no accounts, no cloud, no automatic updates.

---

## 2. Locked product decisions

| # | Topic | Decision |
|---|--------|----------|
| 1 | Updates | **No automatic updates.** Users install / replace builds manually. No update checker phoning home. |
| 2 | Receive policy | **Default: confirm before receive.** Incoming files/photos require explicit accept. |
| 3 | Trust model (v1) | **Discovery is enough to chat.** Any Mac that appears on the LAN device list can open a 1:1 session without pairing codes. |
| 4 | Group chat | **v1: firmly out of scope.** 1:1 only. |
| 5 | Message history | **Persist on device indefinitely** until the user deletes it. **Uninstall does not wipe history.** History lives in a durable app data directory; user must use in-app delete (or manual folder delete) to clear. Received files stay under the user’s save location unless deleted. |
| 6 | Product name | **jotainchatttttttt** |
| 7 | Platforms | **macOS only.** No Windows / Linux in this plan. |

## 3. Goals and non-goals

### Goals (v1)

- Run entirely on the local network (through the Wi‑Fi router); no internet required for core features.
- **Mac ↔ Mac:** auto-discover peers, 1:1 text chat, send photos and files with progress.
- Default receive confirmation for transfers.
- Local message history that survives app restarts and (by design) survives typical uninstall unless the user clears data.
- No auto-update mechanism; no telemetry / cloud accounts.
- Ship as manual `.app` / `.dmg`.

### Non-goals

- **Windows or other OS clients** (removed from plan).
- Group chat, channels, @mentions.
- Pairing codes, room passwords, enterprise PKI.
- Internet / WAN connectivity, relays, hole-punching.
- End-to-end encryption beyond “trusted LAN” baseline.
- Voice/video calls, stickers, cloud backup.
- Auto-update channels.

## 4. User experience

### 4.1 First launch

1. App starts local services (discovery + accept connections).
2. Prompt for **display name** (editable later in settings).
3. Generate stable **device_id** (UUID) stored in app data; never recycled casually.
4. Show empty device list + short help: same Wi‑Fi, allow **Local Network** if macOS prompts.

### 4.2 Main UI

| Region | Behavior |
|--------|----------|
| Sidebar | Online/offline peers + existing 1:1 threads |
| Main pane | Message history for selected peer; text input |
| Transfer UI | File cards in-thread (name, size, progress, accept/reject/cancel) |
| Settings | Display name, default save directory, history path explanation |

### 4.3 Chat

- Only **1:1**. Selecting a discovered device opens or focuses that thread.
- Text messages append to local history immediately; send over LAN when peer reachable.
- Offline peer: show last-known state; outbox = **best-effort while app running** only. Document: “peer must be online.”

### 4.4 Files / photos

1. Sender attaches file(s) or image → **file offer** message in thread.
2. Receiver → **Accept** or **Reject** (no silent receive).
3. On accept: stream to save directory; progress; verify size/hash.
4. On reject / cancel: terminal state on both sides.
5. Large files use a **separate data plane** so chat stays responsive.

### 4.5 History and uninstall

- **Messages:** SQLite under `~/Library/Application Support/com.jotain.jotainchatttttttt/` (Tauri app data dir).
- **Uninstall:** deleting the `.app` does **not** delete Application Support or Downloads.
- **Delete history:** in-app only (message / thread / all). Optional checkbox “also delete received files” default **off**.

## 5. System architecture (Mac-only)

```
┌──────────────────────────────────────────┐
│  UI (TypeScript + React)                 │
│  devices · threads · messages · transfers│
└──────────────────┬───────────────────────┘
                   │ Tauri IPC + events
┌──────────────────▼───────────────────────┐
│  Core (Rust, single process)             │
│  ┌─────────────┐ ┌─────────────────────┐ │
│  │ discovery   │ │ session registry    │ │
│  │ (UDP)       │ │ (peer → TCP conn)   │ │
│  └─────────────┘ └──────────┬──────────┘ │
│  ┌─────────────┐ ┌──────────▼──────────┐ │
│  │ messaging   │ │ file transfer       │ │
│  │ + SQLite    │ │ workers (chunk I/O) │ │
│  └─────────────┘ └─────────────────────┘ │
│  config · path sanitize · notifications  │
└──────────────────┬───────────────────────┘
                   │ LAN (UDP + TCP)
              Wi‑Fi router / switch
                   │
              peer Mac (same stack)
```

### 5.1 Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Shell | Tauri 2 (macOS) | Native feel, small footprint |
| Core | Rust | Reliable networking + file I/O |
| UI | React + TypeScript | Chat UI iteration |
| Local DB | SQLite | Durable history |
| Package | Manual `.app` / `.dmg` | No auto-update |

### 5.2 Process model

- **One process** owns: UDP discovery, TCP accept, per-peer control connections, transfer workers, SQLite.
- UI is display + input only; **no wire protocol in JS**.
- Background tasks: discovery ticker, keepalive, transfer read/write loops, DB writer.

### 5.3 Connection topology (critical)

For each remote `device_id`, maintain **at most one logical control session**.

| Rule | Why |
|------|-----|
| Prefer **dialer = lower UUID** when both sides try to connect | Avoids double TCP sessions / split brain |
| Rebind session when peer IP changes (DHCP / sleep / Wi‑Fi roam) | Common on laptops |
| Map thread UI to `device_id`, never to raw IP alone | IP is unstable |

### 5.4 Threading / async model (recommended)

- Tokio (or equivalent) runtime in Rust.
- **Single writer** for SQLite (channel → writer task) to avoid `SQLITE_BUSY` races.
- Transfer workers report progress via channel → emit Tauri events to UI (throttled).
- Never block the async runtime on large `std::fs` without `spawn_blocking`.

### 5.5 macOS-specific integration

- **Local Network** privacy permission (macOS 14+): failure mode = empty peer list.
- App Transport / firewall: user may need to allow incoming connections on first TCP listen.
- Sleep/wake: cancel stale sockets; restart discovery announce on wake.
- Notifications (optional v1): new message / file offer when window unfocused.

## 6. Network design

### 6.1 Discovery

- **UDP broadcast** on a fixed discovery port.
- Payload (versioned JSON): `protocol_version`, `device_id`, `display_name`, `os: "macos"`, `control_port`, `capabilities`.
- Interval announce + listen; peer offline after several missed intervals.
- Ignore packets from self (`device_id` equal).
- **Trust:** discovery ⇒ allowed to open chat (product decision).

### 6.2 Control plane

- TCP listen on `control_port`.
- Length-prefixed frames (`u32` BE length + UTF-8 JSON body). Max frame size hard cap.
- Message types: `hello`, `text`, `ack` (optional), `file_offer`, `file_accept`, `file_reject`, `file_cancel`, `ping`/`pong`.
- Keepalive; dead connection → rediscovery reconnect.

### 6.3 Data plane (files)

1. `file_offer` on control (`file_id`, basename, size, mime, `token`).
2. Receiver Accept → data TCP; prove `file_id` + `token`.
3. Chunked body + final hash; incomplete files as `*.partial` then rename on success.
4. Reject/cancel stops I/O and deletes partials.

Default save root: `~/Downloads/jotainchatttttttt`.

### 6.4 Security posture (v1)

- Threat model: privacy vs cloud IMs, **not** hostile LAN attacker.
- Hygiene: basename-only paths, frame size caps, no auto-execute received files.
- No TLS required for v1 by product trust model.

### 6.5 Protocol versioning

- `hello.protocol_version` required.
- Mismatch → UI error; **no auto-update**, manual matching build.

## 9. Architecture review

### 9.1 What is sound

| Area | Assessment |
|------|------------|
| UI / core split | Good — protocol stays in Rust |
| Separate file data plane | Good — large files won’t freeze text |
| `device_id` as identity | Good — survives DHCP better than IP |
| Confirm-to-receive | Good — reduces drive-by writes |
| Durable SQLite + no uninstall wipe | Matches product; must document |
| Single-process ownership of sockets | Enough for 1:1 LAN |

### 9.2 Structural risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| Dual dial → two control connections | **High** | Connection arbitration (§5.3) |
| Discovery IP stale after sleep/roam | **High** | Latest announce; reconnect on write fail |
| Chat and file share one mutex/runtime stall | **High** | Async + `spawn_blocking` for disk |
| SQLite concurrent writers | **Med** | Single writer task / WAL |
| Trust-on-discovery | **Accepted** | Product choice; sanitize paths |
| Partial file left forever | **Med** | `*.partial` + cleanup |
| Message ID collision | **Med** | UUIDv4 per message |
| Port already in use | **Med** | Clear error; announce actual port |
| macOS Local Network deny | **High (UX)** | Empty-state education |
| AP/client isolation | **High (env)** | Help text after zero peers |
| History grows unbounded | **Low/Med** | Clear-all exists |
| Token in file_offer guessable | **Med on hostile LAN** | 128-bit token; short TTL |

### 9.3 Module boundaries

```
src-tauri/src/
  main.rs / lib.rs
  config.rs            # ✅ PR1
  db/                  # PR4+
  discovery.rs         # PR3
  net/
    frame.rs
    session.rs
    messages.rs
    transfer.rs
  fsutil.rs
  app_state.rs
```

## 10. Bug-prone areas (checklist)

### 10.1 Discovery

Self-filter, multi-interface announce, offline debounce, stale peer after leave.

### 10.2 Session / TCP

Dual dial, half-open after sleep, frame length DoS, JSON stream desync, reconnect replay → idempotent `message_id`.

### 10.3 Messaging / DB

Persist vs send order, clear thread while transfer runs, SQLITE_BUSY, huge history load.

### 10.4 File transfer

Late accept after cancel, path traversal, overwrite, disk full, hash on main thread, progress flood, token reuse.

### 10.5 macOS lifecycle

Local Network deny, firewall one-way, App Nap, quit mid-transfer, multi-instance, Gatekeeper.

### 10.6 Product footguns

Uninstall does not wipe (document), clear history vs files, display name collision → show device_id suffix.
